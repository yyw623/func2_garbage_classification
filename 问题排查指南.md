# 超分辨率模型推理失败 - 问题排查指南

## 现象
手机安装 APK 后，体验区→高级区→自定义模型→选图→开始超分，提示"推理失败"或报错。

## 上次 Logcat 错误
```
CustomModelActivity: Error loading model: 6
```
错误码 6 = 模型文件无法被 MindSpore Lite runtime 加载。

---

## 按可能性排序的排查清单

### 高可能性

**1. MindSpore Lite 1.1.0 太旧，不兼容模型格式**

原因：项目使用的 mindspore-lite AAR 是 1.1.0（2021 年发布），模型虽然用同版本的 converter_lite 转换，但 converter 生成的内部格式可能 runtime 读不懂（ONNX opset 10 的部分算子翻译不完整）。

验证方法：安装 APK → 操作推理 → Android Studio Logcat 搜 `custommodel` → 看是否还是 `Error loading model: 6`。

解决方案：
- 方案A：升级 mindspore-lite AAR 到 2.x（需从华为云下载对应 AAR，或用 Maven 依赖）
- 方案B：换一个模型（见文末）

---

### 中可能性

**2. Java 预处理代码写错了（Y 通道提取/归一化不对）**

位置：[CustomModelExecutor.java](custommodel/src/main/java/com/mindspore/custommodel/CustomModelExecutor.java) 第 130-155 行

当前逻辑：
```java
// RGB → Y
float y = 0.299f * r + 0.587f * g + 0.114f * b;
y /= 255.0f;  // 归一化到 [0, 1]
```

可能问题：
- 归一化方式不符合模型训练时的设置（该模型可能用的是 ImageNet 均值/标准差）
- 像素顺序（NCHW vs NHWC）可能不对

验证方法：加载模型不报错但输出图片全黑/花屏/异常 → 就是预处理或后处理写错了。

解决方案：检查 ONNX 模型训练时的预处理参数，修改归一化逻辑。

---

**3. Java 后处理代码写错了（YCbCr→RGB 还原不对）**

位置：[CustomModelExecutor.java](custommodel/src/main/java/com/mindspore/custommodel/CustomModelExecutor.java) 第 162-190 行

当前逻辑：
```java
// 从输出 Y 反归一化
int y = Math.round(outputY[i] * 255.0f);
// 从放大原图取 RGB，算 CbCr
// YCbCr → RGB
newR = y + 1.402f * (cr - 128);
newG = y - 0.344136f * (cb - 128) - 0.714136f * (cr - 128);
newB = y + 1.772f * (cb - 128);
```

可能问题：
- 原图上采样用的是简单双线性插值（`Bitmap.createScaledBitmap`），质量低但不是 bug
- YCbCr 转换公式精度问题

验证方法：输出图片颜色偏绿/偏紫/饱和度过高 → 后处理公式问题。

---

### 低可能性

**4. 模型路径或打包问题**

位置：APK 中 `assets/model/super_resolution.ms`（241KB）

验证方法：用 Android Studio 的 APK Analyzer（Build → Analyze APK）打开 APK，确认 `assets/model/super_resolution.ms` 存在且大小正常。

---

**5. 手机内存不足**

Sub-Pixel CNN 模型很小（241KB），内存需求极低，可能性极小。

---

**6. 模型文件损坏**

converter 转换过程中出错导致 .ms 文件内部损坏。可能性低（converter 报告了 SUCCESS）。

---

## 排查步骤建议

```
第1步：装 APK，看 Logcat 错误码
  ├── 还是 error 6 → 升级 MindSpore Lite（见第 1 项）
  ├── 加载成功但图片异常 → 检查预处理/后处理（见第 2、3 项）
  └── 闪退/其他错误 → 看异常堆栈，针对性修复

第2步：如果升不到新版 AAR
  └── 换模型方案见下方
```

---

## 如果当前模型搞不定，换模型的建议

当前模型：ONNX Sub-Pixel CNN（opset 10, DepthToSpace 算子）

可选替代：
- ONNX 模型库中找一个 opset 更低的超分模型
- 或者用 MindSpore 官方 `download.mindspore.cn` 上已有的 .ms 模型（虽然没有超分专项，但可以用图像分割、风格迁移等其他模型完成作业功能展示）
